schmea
    数据表所属 schema

table_name
    要操作的数据表

full_table_name
    包含前缀的数据表名称
    如果指定了 full_table_name，则 QTable_Base 不会根据全局数据表前缀在 table_name 之前添加前缀

pk
    主键字段
    如果有多个主键字段，则以逗号分隔或使用数组表示

belongs_to
has_one
has_many
many_to_many
    定义数据表关联
    关联的详细定义，参考 TableDataGateway Links.txt 文件

created_time_fields
    创建记录时，要填充当前时间的字段

updated_time_fields
    更新记录时，要填充当前时间的字段

CRUD 对关联表条件的支持：

find: orders belongs_to members
    SELECT orders.* FROM orders

    LEFT JOIN members
    ON members.member_id = orders.user_id

    WHERE orders.created < 1191196800
        AND members.username = 'tira'


find: orders belongs_to members AND orders has_many order_items
    SELECT orders.* FROM orders

    LEFT JOIN members
    ON members.member_id = orders.user_id

    LEFT JOIN order_items
    ON order_items.order_id = orders.order_id

    WHERE orders.created < 1191196800
        AND members.username = 'tira'
        AND order_items.product_id = 22508

find: products many_to_many tags
    SELECT products.* FROM products

    LEFT JOIN products_has_tags
    ON products_has_tags.product_id = products.product_id
    LEFT JOIN tags
    ON products_has_tags.tag_id = tags.tag_id

    WHERE products.product_id = 13111
        AND tags.tagname = '连衣裙'


update: products many_to_many tags
    UPDATE products
    SET vip_price = 123
    WHERE products.product_id = 13111

    要转变为：

    UPDATE products
        [[
        , products_has_tags, tags
        ]]

    SET vip_price = 123

    WHERE
        products.product_id = 13111
        [[
        AND tags.tagname = '连衣裙'
        AND products_has_tags.product_id = products.product_id
        AND products_has_tags.tag_id = tags.tag_id
        ]]
